## Documentation Standards

### Format

All documentation is written in Markdown format (.md files) for:
- Easy reading in GitHub
- Version control friendly
- Portable across platforms
- Simple syntax

### Structure

Each document follows this structure:
- Clear title and description
- Table of contents (for longer docs)
- Logical section organization
- Code examples where appropriate
- Cross-references to related docs

### Maintenance

Documentation is updated when:
- New features are added
- Bugs are fixed that affect usage
- User feedback indicates confusion
- API or architecture changes
- New best practices emerge

## Contributing to Documentation

Documentation improvements are always welcome!

### How to Contribute

1. **Fix typos or errors**
   - Fork repository
   - Edit the .md file
   - Submit pull request

2. **Add clarifications**
   - Identify confusing sections
   - Add examples or explanations
   - Submit pull request

3. **Create new guides**
   - Identify missing topics
   - Write comprehensive guide
   - Follow existing format
   - Submit pull request

4. **Update screenshots**
   - Capture current UI
   - Save as PNG or JPG
   - Place in `docs/images/` (if created)
   - Update references

### Documentation Style Guide

- Use clear, concise language
- Write in present tense
- Use active voice
- Include code examples
- Add keyboard shortcut references
- Link to related documents
- Keep sections focused
- Use proper Markdown formatting

## Getting Help

Can't find what you're looking for?

1. **Search all documentation**
   - Use GitHub's search feature
   - Search for keywords across all .md files

2. **Check GitHub Issues**
   - Someone may have asked your question
   - Search both open and closed issues

3. **Ask in Discussions**
   - Start a discussion on GitHub
   - Community may have answers

4. **Open an Issue**
   - If documentation is unclear
   - If something is missing
   - If you found an error
